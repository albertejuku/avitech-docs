# Payments Tab

Found on the [Subscriber Details](details.md) page, the Payments tab shows the complete payment history for this subscriber.

*Screenshot: Payments tab — add image here*

## Payment History Table

| Column | Description |
|---|---|
| Date | Date and time the payment was recorded |
| Amount | Amount paid in KES |
| Provider | Payment channel — M-Pesa, Cash, or Bank Transfer |
| Reference | Transaction reference / receipt number |
| Method | Specific payment method used |
| Status | Completed, Pending, or Failed |

The provider is shown as a colored chip:

- **M-Pesa** — green
- **Cash** — amber
- **Bank Transfer** — blue

## Pagination

The table shows 8 payments per page. Use the navigation buttons at the bottom to move between pages. The total transaction count is shown above the table.

## Recording a New Payment

To record a payment for this subscriber, go to the **[Payments](../payments/index.md)** section and use **Record Payment** — search for the subscriber by name or phone to link the payment to their account.

Alternatively, use **Renew / Change Plan** on the [Subscription Tab](subscription-tab.md) which records a payment and extends the subscription in one step.

!!! tip
    The Payments tab is read-only. All payment actions (record, redeem, STK push) are managed from the main [Payments page](../payments/index.md).
